## 第 1 步：准备两个东西

告诉用户需要去获取以下两个凭证，拿到后告诉你：

**1.1 DeepSeek API Key**

1. 打开 https://platform.deepseek.com/ 注册/登录
2. 进「API Keys」→ 点「Create new secret key」
3. 复制 Key（格式 `sk-xxxx`）
4. **需要先充值才能用**（最低 1 元起充，充 1-5 元够用很久）

**1.2 QQ 邮箱授权码（容易卡住，仔细看）**

1. 打开 https://mail.qq.com/ 登录你的 QQ 邮箱
2. 点页面顶部的「**设置**」（在邮箱页面顶部，不是 QQ 主面板）
3. 左侧菜单选「**账户**」
4. 往下翻找到「**POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务**」
5. 找到「**IMAP/SMTP服务**」这一行，点右边「**开启**」
6. 按提示用手机发短信验证（免费）
7. 拿到一个 **16 位字符串**，这就是授权码，**立刻复制保存**
8. **这不是你的 QQ 密码，是授权码。**

确认用户两个都准备好了，进入第 2 步。

## 第 3 步：添加 Secrets（自己定义的秘密变量）

告诉用户去自己的 Fork 仓库里操作：

1. 进入自己 Fork 的仓库（`你的用户名/github-blindbox`）
2. 点顶部的 **Settings** → 左侧 **Secrets and variables** → **Actions**
3. 点 **New repository secret**，依次添加 3 个：

| Secret 名称 | 填什么 |
|-------------|--------|
| `ANTHROPIC_AUTH_TOKEN` | DeepSeek 的 `sk-xxxx` |
| `ANTHROPIC_BASE_URL` | `https://api.deepseek.com/v1` |
| `ANTHROPIC_MODEL` | `deepseek-chat` |
| `QQ_EMAIL` | 你的 QQ 邮箱（如 `123456@qq.com`） |
| `QQ_SMTP_AUTH_CODE` | 刚才拿到的 16 位授权码 |

**注意：名称必须一字不差，包括大小写和下划线。**

加上去了告诉我。

## 第 5 步：自定义（选做）

恭喜用户收到了第一封邮件，然后**主动问两个问题**：

**5.1 筛选标准要不要调？**

默认筛选的是「独立开发者、vibecoding 实践者」视角，会过滤掉底层技术库、算法、Web 框架。

问问用户：「你平时主要关注什么方向？可以跟我说，我告诉你怎么改。」

常见方向：
- A. 默认就行（独立开发者、产品思维）
- B. AI 技术研究（关注 RAG/Agent/训练等技术细节）
- C. 设计/创意类（关注视觉、交互、新奇项目）
- D. 后端/系统类（关注数据库、分布式、性能优化）
- E. 其他（用户自己说）

**告诉他改哪里：** 在自己的 Fork 仓库里打开 `scripts/github-digest.js`，找到「阅读者画像」那段文字，改成自己的描述，然后点右上角的 **Commit changes**。

**5.2 推送时间要不要调？**

告诉用户：
- 船长每天 **05:00 抓数据 commit 到仓库**
- **推送时间必须在 07:00 之后**（因为 GitHub Actions 可能有延迟，太早可能拉到旧数据）
- 建议时间：07:00 / 08:30 / 12:00 / 18:30（默认） / 20:00 / 22:00

**告诉他改哪里：** 在自己的 Fork 仓库里打开 `.github/workflows/digest.yml`，找到 `cron` 那行，改成对应的时间。

UTC 换算方法：`UTC = 北京时间 - 8`。比如想 20:00 收到，就填 `0 12 * * *`。
